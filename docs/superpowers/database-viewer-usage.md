# Просмотрщик баз данных — как пользоваться

Встроенный в этот форк Zed инструмент для быстрого просмотра баз данных PostgreSQL:
дерево коннекшенов, структура таблиц, просмотр строк с сортировкой/фильтрами/пагинацией,
SQL-вкладка и MCP-сервер для AI-агентов.

## Открыть панель

Иконка базы данных в левом доке открывает панель «Databases». Если её не видно,
вызовите действие `database_panel::ToggleFocus` из палитры команд (Cmd+Shift+P →
«database panel»).

## Добавить подключение

1. Кнопка `+` в шапке панели → форма подключения.
2. Заполните: имя (уникальное), host, port (по умолчанию 5432), база, пользователь, пароль.
3. «Test Connection» проверяет доступность; «Save» сохраняет.

Параметры подключения пишутся в `settings.json` (секция `database.connections`),
**пароль хранится только в macOS Keychain** и на диск не попадает.

Пример секции настроек:

```jsonc
"database": {
  "page_size": 100,            // строк на страницу
  "query_timeout_seconds": 30, // таймаут запроса
  "mcp_max_rows": 200,         // лимит строк в ответе MCP-инструмента
  "connections": [
    { "name": "local-shop", "host": "127.0.0.1", "port": 5432, "database": "shop", "user": "postgres" }
  ]
}
```

Коннекшены можно править и руками в `settings.json`; пароль после этого задаётся
через «Edit Connection» в контекстном меню (правый клик по коннекшену).

## Просмотр данных

- Клик по коннекшену раскрывает базы → схемы → таблицы (загрузка ленивая).
- Клик по таблице открывает вкладку с двумя режимами:
  - **Data**: первые N строк; клик по заголовку колонки циклически сортирует
    (↕ → ↓ → ↑); кнопка «+ Filter» добавляет фильтр по полю
    (`=`, `≠`, `>`, `<`, `contains`, `is null`); стрелки внизу листают страницы.
  - **Structure**: колонки (тип, nullable, default, PK, FK) и индексы.

## SQL-запросы

Правый клик по коннекшену или базе → «New SQL Query» открывает вкладку с редактором
SQL и таблицей результатов. Запуск — **Cmd+Enter** (macOS) / **Ctrl+Enter** (Linux).
Сессии UI открыты в режиме read-write, поэтому из SQL-вкладки можно выполнять и
`UPDATE`/`INSERT`/`DELETE`, а не только `SELECT`.

## Доступ для AI-агентов (MCP)

Отдельный бинарь `zed-database-mcp` даёт агентам доступ к тем же базам **только на
чтение** (сессии MCP жёстко read-only, что бы агент ни прислал). Он использует те же
коннекшены из `settings.json` и пароли из Keychain.

Инструменты: `list_connections`, `list_tables`, `describe_table`, `run_query`.

### Подключить к агенту Zed

Соберите бинарь и пропишите его как context server:

```bash
cargo build --release -p database_mcp   # → target/release/zed-database-mcp
```

```jsonc
// settings.json
"context_servers": {
  "zed-database": { "command": "/path/to/zed/target/release/zed-database-mcp", "args": [] }
}
```

### Подключить к Claude Code

```bash
claude mcp add zed-database /path/to/zed/target/release/zed-database-mcp
```

## Что пока не поддерживается (планы)

- Другие СУБД (MySQL/MariaDB, SQLite) — архитектура готова к добавлению.
- SSH-туннели (пока — вручную `ssh -L`), обязательный TLS/sslmode.
- Редактирование данных прямо в таблице (правка ячеек, добавление строк) — этап 2.
- Экспорт результатов и сохранение `.sql`-файлов.
