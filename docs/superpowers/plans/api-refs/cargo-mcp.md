# Zed workspace/Cargo conventions for new crates + in-repo MCP (context_server) protocol types

# Topic A — Workspace / Cargo conventions

## 1. Root `Cargo.toml` (`/Users/user/zed/Cargo.toml`)

**Members: explicit list, NOT a glob.** `[workspace] resolver = "2"` and every crate is listed alphabetically by path (lines 2–258), ending with `extensions/*` and `tooling/*` entries; then `default-members = ["crates/zed"]` (line 259). Adding a new crate requires TWO edits to the root manifest:

1. Add `"crates/your_crate",` to `members` (alphabetical position).
2. Add a `[workspace.dependencies]` entry so other crates can use `your_crate.workspace = true`:

```toml
# /Users/user/zed/Cargo.toml, [workspace.dependencies] starts line 265;
# workspace-member crates come first (lines 271+), e.g.:
csv_preview = { path = "crates/csv_preview" }
context_server = { path = "crates/context_server" }
# external deps follow later in the same section, e.g.:
anyhow = "1.0.86"                                        # line 515
serde = { version = "1.0.221", features = ["derive", "rc"] }  # line 747
```

**`[workspace.package]`** (lines 261–263):

```toml
[workspace.package]
publish = false
edition = "2024"
```

Leaf crates inherit via `publish.workspace = true` and `edition.workspace = true`.

**Lints** (lines 1020–1061): root defines `[workspace.lints.rust]` (`unexpected_cfgs = { level = "allow" }`) and `[workspace.lints.clippy]` (`dbg_macro = "deny"`, `todo = "deny"`, `declare_interior_mutable_const = "deny"`, `redundant_clone = "deny"`, `disallowed_methods = "deny"`, `style = { level = "allow", priority = -1 }`, plus several `allow`s). Every leaf crate opts in with:

```toml
[lints]
workspace = true
```

Caveat: `types::*` is fully `pub`, but the JSON-RPC envelope types `Response`/`CspResult`/`Client` are `pub(crate)` — outside the crate you get `Request`, `RequestId`, `AnyRequest`, and the error-code consts, but not `Response`. A server outside the crate either defines its own tiny response envelope or uses the in-crate `listener::McpServer`.
