**Step 2: Add `prune_undo_history` free function to items.rs**

In `crates/editor/src/items.rs`, add a new free function `prune_undo_history` in the `impl Editor` block at the bottom (after `restore_undo_history`, around line 1680). The function follows the exact same settings-gate-before-background-spawn pattern as `write_undo_history`:

```rust
fn prune_undo_history(workspace_id: WorkspaceId, cx: &App) -> Task<()> {
    use settings::Settings as _;
    if !PersistentUndoSettings::get_global(cx).enabled {
        return Task::ready(());
    }

    cx.background_spawn(async move {
        let paths = match DB.get_undo_history_paths(workspace_id) {
            Ok(paths) => paths,
            Err(err) => {
                log::warn!("Failed to query undo history paths for pruning: {err}");
                return;
            }
        };

        for abs_path in paths {
            if smol::fs::metadata(&abs_path).await.is_err() {
                DB.delete_undo_history(workspace_id, Arc::from(abs_path.as_path()))
                    .await
                    .log_err();
                let blob_path = blob_path_for(&abs_path);
                if let Err(err) = smol::fs::remove_file(&blob_path).await {
                    if err.kind() != std::io::ErrorKind::NotFound {
                        log::warn!("Failed to remove undo history blob {blob_path:?}: {err}");
                    }
                }
            }
        }
    })
}
```

IMPORTANT: This is a free function (not a method on `&self`), placed OUTSIDE the `impl Editor` block at the module level, similar to `blob_path_for` and `compute_content_hash`. It takes `workspace_id` and `cx` directly. The reason is that `cleanup` is a static method on `SerializableItem` -- it has no `&self`. Place it near the other undo-history free functions at the top of the file.

Key implementation notes:
- Settings gate happens on the foreground thread BEFORE `cx.background_spawn` (avoids panic).
- `smol::fs::metadata` returns `Err` when file does not exist -- treat any error as "absent."
- `DB.delete_undo_history` takes `Arc<Path>` -- convert with `Arc::from(abs_path.as_path())`.
- `blob_path_for` is already a free function in the same file -- call it directly.
- `NotFound` on blob removal is benign (blob may not exist if write was interrupted).
- `.log_err()` on DB delete is sufficient -- pruning is best-effort background work.

**Step 3: Wire `prune_undo_history` into `Editor::cleanup`**

Modify the existing `cleanup` method in `impl SerializableItem for Editor` (line 1154) to also invoke `prune_undo_history`. The current implementation is a one-liner that returns `workspace::delete_unloaded_items(...)`. Change it to combine both tasks:

```rust
fn cleanup(
    workspace_id: WorkspaceId,
    alive_items: Vec<ItemId>,
    _window: &mut Window,
    cx: &mut App,
) -> Task<Result<()>> {
    let editor_cleanup =
        workspace::delete_unloaded_items(alive_items, workspace_id, "editors", &DB, cx);
    let prune = prune_undo_history(workspace_id, cx);
    cx.background_spawn(async move {
        editor_cleanup.await?;
        prune.await;
        Ok(())
    })
}
```

Note: `editor_cleanup` returns `Task<Result<()>>` and must propagate errors. `prune` returns `Task<()>` (logs errors internally, does not propagate) -- just `.await` it without `?`.
