# Save Session Context

Document everything we did in this session to `_context.md`.

## What to document:

1. **Session Summary**: What we were trying to accomplish
2. **Files Changed**: List every file we created or modified with line numbers
3. **What Changed**: Explain what was changed in each file and why
4. **Problems Encountered**: Any bugs or issues we ran into
5. **How We Fixed Them**: The solutions we implemented
6. **Current State**: What's working now and what might still need attention

## Format:

```markdown
# Session: [Brief description]
Date: [Today's date]

## Goal
[What we were trying to accomplish]

## Files Changed

### Created
- `path/to/file.ts` - [What it does]

### Modified
- `path/to/file.ts:123-145` - [What changed and why]

## Problems & Solutions

### Problem 1: [Description]
**Issue**: [What went wrong]
**Cause**: [Why it happened]
**Fix**: [How we solved it]

## Current Status
- ✅ [What's working]
- ⚠️ [What needs attention]
- 📝 [Notes for next time]
```

## Instructions:

Write a comprehensive but concise summary to `_context.md`. Be specific with file paths and line numbers. Explain the "why" not just the "what".
